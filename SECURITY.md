# Security And Privacy Reporting

Please do not publish real secrets, private prompts, private account data, wallet
material, viewing keys, seed phrases, API credentials, customer data, source code,
security payloads, invoice data, bank details, production deploy commands, or private
workflow payloads in issues, pull requests, examples, or screenshots.

If you find a privacy or safety issue in this repository, open a minimal GitHub issue
that describes the class of problem without including sensitive data. If private
coordination is needed, contact:

dev@alfred-ai.app

## Scope

In scope:

- unsafe fixture fields,
- redaction bypasses,
- examples that reveal private material,
- examples that imply custody or live-funds control,
- examples that imply autonomous external sends, production deploys, security
  remediation, spend approval, or business-record mutation without explicit approval,
- validator failures that allow unsafe public receipts,
- documentation that encourages publishing raw prompts, secrets, source code, security
  payloads, finance data, or private business records.

Out of scope:

- live chain exploits,
- custody incidents,
- wallet compromise reports,
- token or bridge incidents,
- production payment execution.
- production deployment authority,
- production security-remediation authority,
- customer-data incidents in systems outside this public specification repository.

This repository is a simulated public specification package, not Aweb Mission Control
Cloud and not a production wallet, custody, token, bridge, trading, payment,
deployment, security-remediation, accounting, or communication-sending system.
