# Security And Privacy Reporting

Please do not publish real secrets, private prompts, private account data, wallet
material, viewing keys, seed phrases, API credentials, customer data, or private
workflow payloads in issues, pull requests, examples, or screenshots.

If you find a privacy or safety issue in this repository, open a minimal GitHub issue
that describes the class of problem without including sensitive data. If private
coordination is needed, contact:

dev@alfred-ai.app

## Scope

In scope:

- unsafe fixture fields,
- redaction bypasses,
- examples that reveal private material,
- examples that imply custody or live-funds control,
- validator failures that allow unsafe public receipts,
- documentation that encourages publishing raw prompts, secrets, or private payloads.

Out of scope:

- live chain exploits,
- custody incidents,
- wallet compromise reports,
- token or bridge incidents,
- production payment execution.

This repository is a simulated public-good seed package, not a production wallet,
custody, token, bridge, trading, or payment system.
