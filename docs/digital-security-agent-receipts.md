# Digital Security Agent Receipts

This note describes the simulated digital-security workflow fixture in:

`fixtures/digital-security-workflow-receipt.example.json`

## Purpose

The fixture shows how an agent receipt can preserve evidence for AI-assisted digital-security and internet-freedom workflows without exposing sensitive users, sources, locations, credentials, private notes, raw prompts, or operational details.

The intended OTF angle is narrow:

- open-source receipt format for accountable AI-assisted workflows,
- privacy-aware redaction boundary,
- human review before publication, outreach, escalation, or operational recommendation,
- explicit denial of offensive automation and sensitive-user data collection.

## What The Receipt Records

- scoped authority before the workflow,
- public-safe research context,
- denied actions around intrusion, exfiltration, bypassing access controls, and at-risk-user contact,
- evidence artifacts with sensitive context redacted,
- review state requiring a human before any downstream action.

## Boundary

This is not surveillance tooling, offensive automation, vulnerability exploitation, target enrichment, private-identity publication, or operational security advice.

All example values are simulated placeholders. The fixture does not process real sensitive-user data and does not contact or identify at-risk people.

## Why It Matters

AI agents can help digital-security teams with triage, documentation, research, and support workflows, but high-risk environments cannot accept opaque tool use or unsafe logs. Receipts make the authority, action, evidence, redaction, and recovery state explicit.
